Loop/While

Literals

Literals in Loop/While programs can be variables or constants.

Variables

  • Variables start with the letter x followed by a natural number (x0, x1, x2, ...).
  • x0 usually serves as the return variable and should hold the result of the program.
  • x0, x1, x2, ... are initialized from the program's parameters.
  • Unused variables are initialized to 0 by default.
  • If a variable would be decremented below 0, it is set to 0 instead.

Constants

  • Constants are natural numbers (0, 1, 2, ...).
  • Only non-negative values are allowed.

Keywords

The following keywords control the program's flow:

  • Loop: Initiates a fixed-number iteration block.
  • While: Starts a conditional iteration block.
  • Do: Indicates the start of the block for both Loop and While.
  • End: Marks the end of a Loop or While block.

Use of semicolon

  • Semicolons are used as separators to distinguish between sequential statements within a block.They clarify the sequence of operations when chaining multiple commands.
  • A semicolon is required between statements but not allowed before an End or at the end of the program.

Programs

  1. Assignments:

    Assigns the result of a constant added to or subtracted from a variable to a variable:

    x1 = x1 + 6;
x2 = x1 - 4
  2. Loop Programs:

    Repeats a block of instructions a fixed number of times. This can either be a constant or a variable.

    x0 = x0 + 6;
x1 = x1 + 4;

Loop x1 Do
   x0 = x0 + 1
End;
Loop 3 Do
   x0 = x0 + 2
End
  3. While Programs:

    Repeats a block of instructions as long as a variable is larger than zero.

    x1 = x1 + 4;
x0 = x0 + 6;

While x1 > 0 Do
   x0 = x0 + 1;
   x1 = x1 - 1
End
  4. Nested Loops:

    Multiple loops can be nested:

    x1 = x1 + 5;
x2 = x2 + 2;

While x2 > 0 Do
   x3 = x3 + 2;
   Loop x1 Do
     x4 = x4 + 1
   End;
   x2 = x2 - 1
End

Code Examples

Example 1: This program increments x1 by 1 x2 times.

x1 = x1 + 6;
x2 = x2 + 4;

Loop x2 Do
   x1 = x1 + 1
End

Example 2: This program decrements x2 and increments x1 until x2 = 0.

x1 = x1 + 6;
x2 = x2 + 4;

While x2 > 0 Do
   x1 = x1 + 1;
   x2 = x2 - 1
End

Goto

Literals

Literals in GOTO programs can be variables or constants.

Variables

  • Variables start with the letter x followed by a natural number (x0, x1, x2, ...).
  • x0 usually serves as the return variable and should hold the result of the program.
  • x0, x1, x2, ... are initialized from the program's parameters.
  • Unused variables are initialized to 0 by default.
  • If a variable would be decremented below 0, it is set to 0 instead.

Constants

  • Constants are natural numbers (0, 1, 2, ...).
  • Only non-negative values are allowed.

Markers

Markers (M1, M2, ...) label instructions and facilitate jumps within the program.

Keywords

GOTO programs use the following keywords to define program flow and structure:

  • Goto: Performs an unconditional jump to a specified marker.
  • If: Used for conditional jumps; checks if a variable equals a constant before jumping.
  • Then: Completes the condition in a conditional jump statement.
  • Halt: Terminates the program.

Use of semicolon

  • Semicolons act as separators between instructions, ensuring that each command is distinct.
  • The last line of a program should not end with a semicolon.

Program Structure

A GOTO program is a sequence of instructions that operate on variables, with flow control achieved through markers (M1, M2, ...).

  1. Assignments:

    Assigns the result of a constant added to or subtracted from a variable to a variable:

    M1: x1 = x2 + 0;
M2: x3 = x4 - 1
  2. Conditional Jumps:

    Check if a variable equals a constant and jump to a marker:

    M1: x1 = x1 + 6;
M2: x0 = x0 + 1;
M3: If x1 = 6 Then Goto M5;
M4: x0 = x0 + 10;
M5: Halt
  3. Unconditional Jumps:

    Directly jump to a marker:

    M1: x0 = x0 + 6;
M2: Goto M4;
M3: x0 = x0 + 4;
M4: Halt

Code Example

Example: To calculate the addition of two variables (Add(x, y) = x + y):

M1: x1 = x1 + 4;
M2: x2 = x2 + 6;

M3: x0 = x1 + 0;
M4: If x2 = 0 Then Goto M8;
M5: x2 = x2 - 1;
M6: x0 = x0 + 1;
M7: Goto M4;
M8: Halt

Tutorial

Working with Projects

Creating a New Project

1. Navigate to the project-tab area.

2. Click the "+" Button to create a new project.

  • A new empty project tab (e.g., "Project 2") will appear.

Switching Between Projects

Simply click on a project tab to switch to that project.

Deleting a Project

To close and delete a project, click the "X" Button on its tab

Initializing and Managing Variables

Adding Variables

1. Navigate to the variable-input section.

2. Click the "+" Button to add a new variable.

  • A new variable will be created with an empty value field.

Setting Variable Values

1. Click inside the value field of the variable.

2. Type the desired value.

Renaming Variables

The system uses variable names in the format xN, where x is fixed and N is an integer (e.g., x3 or x4).


1. Double-click on the number in the variable name.

2. Enter a new number to replace the old one.

  • Note: You cannot create entirely new names (e.g., "example-variable").
    You can only modify the numeric value after x.

Deleting Variables

Click the "X" Button next to a variable to delete it.

Writing Code

1. Navigate to the Code Input section.

2. Enter your program using the syntax of the LOOP/WHILE or GOTO language.

Executing Programs

Executing Your Code

1. Click the "Run Button"

2. Observe the output:

  • Success: A success message and the final values of all variables if the program runs correctly.
  • Error: An error message with details, including the type of error and the line number, if issues are detected.

Stopping Execution

Click the "Stop Button" to halt program execution immediately.

Programs that run for longer than 15 seconds will stop automatically.

Debugging Programs

Step-by-Step Execution

1. Click the "Debug Button" (blue with a bug icon) to enter debug mode.

2. Use the "Next Button" to execute the program line by line.

3. Use the "Back Button" to step backward if you need to review previous states.

Monitoring Variables in Debug Mode

While debugging, you can view the real-time values of variables as they update with each step.
